## 📦 模块概览

本模块包含 **3 个数据面板** 和 **1 个定时通知脚本**，覆盖世界杯小组赛全部 72 场比赛。

| 组件 | 脚本 | 刷新间隔 | 功能 |
|:---:|:---:|:---:|:---|
| 🏟️ 最近赛程 | `worldcup_schedule.js` | 2 分钟 | 昨日/今日/明日比赛，实时比分 |
| 🏆 分组排名 | `worldcup_standings.js` | 5 分钟 | 12 组完整积分榜 |
| 📊 统计信息 | `worldcup_stats.js` | 10 分钟 | 71 人进球榜 + 50 人助攻榜 |
| 🔔 赛事通知 | `worldcup_notify.js` | 10 分钟 | 赛前提醒 / 进球通知 / 赛后比分 |

## 🔧 安装方法

### 一键安装

在 Surge 中点击以下链接，或直接复制模块地址添加到 Surge：

```
https://raw.githubusercontent.com/GitHubWBB/Surge/main/fifa2026_v2/2026_WorldCup_v2.sgmodule
```

### 可配置参数

| 参数 | 默认值 | 说明 |
|:---:|:---:|:---|
| `api_key` | 内置默认 | football-data.org API Key（可在 [官网](https://www.football-data.org/) 免费注册获取） |
| `remind_before` | `30` | 赛前提醒提前分钟数 |

## 🔔 赛事通知

> 每 10 分钟执行一次，支持三种通知类型。

| 通知类型 | 触发条件 | 示例 |
|:---:|:---|:---|
| 📋 每日赛程 | 北京时间 8:00 自动推送 | `⚽ 今日赛程 · 今日4场 · [A组] 03:00 ...` |
| ⏰ 赛前提醒 | 开赛前 30 分钟（可配置） | `⚽ 即将开赛 · A组 · 🇲🇽墨西哥 vs 🇰🇷韩国` |
| ✅ 赛后比分 | 比赛结束后即时推送 | `⚽ 比赛结束 · A组 · 墨西哥 2-0 南非 · ⚽ Messi 23' · David 67'` |

**去重机制：** 使用 `$persistentStore` 存储已发送通知的 ID，7 天自动清理，避免重复通知。

## 🌐 数据来源

| 数据 | 来源 | 模式 |
|:---:|:---:|:---:|
| 赛程时间表 | 静态嵌入（72场北京时间） | API 合并比分 |
| 实时比分 | [football-data.org](https://www.football-data.org/) | 实时 API |
| 分组排名 | football-data.org | 实时 API + 静态备用 |
| 进球榜 | football-data.org | 实时 API（71人）+ 静态备用 |
| 助攻榜 | [ESPN](https://www.espn.com/soccer/stats/_/league/fifa.world/season/2026) | 静态（50人） |
| 赛程提醒 | 静态时间表 + API 验证 | 双模式 |
| 赛后通知 | football-data.org（含进球详情） | API 模式 |

## ⚙️ 技术说明

- **双模式架构：** 所有面板优先使用 API 获取实时数据，API 失败时自动回退到内置静态数据
- **北京时间处理：** 所有时间存储为北京时间字符串，`parseBJ()` 转换 UTC，`getBJNow()` 获取当前北京时间
- **API 名称归一化：** `API_MAP` 处理 football-data.org 返回的团队名称与 FLAGS/CN 映射键的差异（如 `Turkey→Turkiye`、`Korea Republic→South Korea`）
- **瑞士国旗：** 因 Surge 无法正确渲染 🇨🇭，统一使用 `[CH]` 标识
- **中文映射：** 71位进球球员 + 50位助攻球员全部配备中文名，50个国家/地区中文名完整覆盖

## 📌 注意事项

- football-data.org 免费版 API 限速 10 次/分钟，建议面板刷新间隔不低于 2 分钟
- 助攻榜使用 ESPN 静态数据（API 仅覆盖约9位助攻者），如需实时更新可替换数据源
- 小组赛阶段（6/12-6/29）结束后，面板将显示所有已结束的比赛
- 进入淘汰赛后，赛程面板需手动更新淘汰赛对阵数据
